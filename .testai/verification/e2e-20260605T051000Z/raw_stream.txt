Loan Processing Pipeline\n\nThe loan lifecycle is managed through status transitions:\n\n```\nDRAFT \u2192 SUBMITTED \u2192 UNDER_REVIEW \u2192 KYC_PENDING \u2192 COMPLIANCE_CHECK \u2192 APPROVED/REJECTED \u2192 DISBURSED\n```\n\nEach step can be handled manually (via API) or automatically via CrewAI agents:\n\n- **Credit Risk Scoring**: ML models score applications \u2014 XGBoost for US (FICO-based), Logistic Regression for India (CIBIL-based)\n- **KYC Verification**: Extract identity data from uploaded document images\n- **AML Compliance**: Screen applicants against sanctions lists (Yente/OpenSanctions) and Wikidata (PEP status)\n- **Provider Search**: Find optimal loan providers with interest rate comparison\n- **CrewAI Decision**: Full end-to-end automated decision using `loan_creation_crew`\n\n### 2. CrewAI Agent System\n\nThe app uses **12 specialized agent crews**, each solving a specific domain:\n\n| Crew | Purpose |\n|------|---------|\n| **System Router** | Routes user queries to the correct specialized crew |\n| **FD Analysis** | Analyzes fixed deposit products and rates |\n| **FD Research** | Searches for current FD rates and news |\n| **FD Database** | Stores and retrieves FD data from database |\n| **FD Visualization** | Generates ECharts visualizations for FD comparisons |\n| **FD Template** | Creates FD certificate templates |\n| **Loan Credit Risk** | Scores loan applications using region-specific ML models |\n| **Loan Creation** | End-to-end loan processing (KYC \u2192 AML \u2192 scoring \u2192 provider) |\n| **AML Execution** | Runs AML compliance checks (sanctions, PEP, adverse media) |\n| **Mortgage Analytics** | US mortgage risk analysis using Fannie Mae models |\n| **TD/FD Creation** | Creates Term Deposit / Fixed Deposit records |\n| **Loan Provider Search** | Searches for optimal loan providers |\n\n### 3. Smart Assistant\n\nA conversational AI interface that:\n- Detects user intent from natural language\n- Routes to the appropriate CrewAI crew\n- Returns formatted responses with visualizations\n\n### 4. Admin Dashboard\n\nFull admin panel at `/admin-panel/` with:\n- User management (list, detail, activate/deactivate)\n- Transaction monitoring (approve/reject loan applications)\n- Analytics dashboard with charts\n- Email campaign creation and sending\n- Database query interface with audit logging\n- Model CRUD (create/read/update/delete any database record)\n- Fixed Deposit management with renewal analysis\n- Session management (admin only)\n- Audit log viewer\n- CrewAI reasoning log viewer\n\n### 5. User Portal\n\nCustomer portal at `/user/` with:\n- Registration with email verification\n- Login/logout with session management\n- Password reset flow\n- Dashboard with loan, FD, and transaction views\n- FD renewal analysis and email notification\n\n### 6. RAG System\n\nRetrieval-Augmented Generation using ChromaDB:\n- Upload policy documents via `rag_upload_app.py`\n- Query documents via `/api/rag-query/`\n- Agents use `rag_policy_search_tool` for
