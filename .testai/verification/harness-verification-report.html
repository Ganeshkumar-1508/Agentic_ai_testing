3.3 Connection reuse

No shared httpx.AsyncClient. 31 distinct client construction sites; only pricing_cache.py reuses one client across its 7-day refresh loop. No limits=, no event_hooks= anywhere — so the redirect-re-validation pattern called out in url_safety.py:20-24 is not actually wired up. trust_env defaults to True, so HTTP_PROXY / HTTPS_PROXY / NO_PROXY from the process env are honored.

3.4 Gaps & risks

  1. No global outbound timeout or rate limit. Each call site picks its own (and several don't). A slow upstream can hang a tool call.
  2. Redirect-time SSRF: web_fetch (web_tools.py:101) and skill install (skill_tools.py:397) follow redirects without re-validating against is_safe_url. A public origin that 302-redirects to http://169.254.169.254/... slips through the pre-flight check.
  3. No shared LLM httpx client — llm.py:308-312 and :473-477 build a fresh openai.AsyncOpenAI (and therefore a fresh httpx client) per call. For high-volume agents this forfeits the SDK's connection pool.
  4. DDG web search has no retry — the HTML endpoint frequently 202/rate-limits.
  5. OSV check and Anthropic model probe are sync via urllib.request. They block an executor thread for up to 10 s on a slow response.
  6. retry_with_backoff is dead code at the call sites that matter — 31+ outbound calls do their own try/except.
  7. Credential sanitization is MCP-scoped only. LLM, GitHub, and webhook errors flow to logs / UI without going through sanitize_error_message.
  8. kanban/dispatcher.py:257-258 swallows all exceptions silently — webhook fan-out failures leave no log, no metric, no retry.
  9. Two Slack code paths with very different reliability profiles: delivery/adapters/slack.py uses the official SDK (internal retry), integrations/slack.py uses raw httpx with no timeout/retry.
  10. is_safe_url is not invoked by most outbound paths — only MCP transports (and a few vision paths) go through it. web_fetch, skill install, custom_notifier, and the kanban webhook fan-out do not.
  11. MCP and LLM circuit breakers have different thresholds (5/60 s vs 3/30 s) with no central policy.
  12. No verify= override — system trust store only, so corporate MITM proxies or a poisoned SSL_CERT_FILE are trusted by default.

3.5 Verdict

PASS The harness reaches the internet through ~31 outbound sites covering LLM, git, chat, image, search, pricing, and operator webhooks. Reliability is layered (circuit breaker + reconnect + health check), safety is layered (SSRF pre-check + error sanitization + trust-level gating for skills), but the layers are inconsistent across call sites. A central HTTP client + a global "go through is_safe_url + sanitize" policy would close the gap.

4.4 Security model (and the gap)

Four trust levels resolved by skills_guard.py:903:

  • builtin — anything in official/*.
  • trusted — openai/skills, anthropics/skills, huggingface/skills (the trusted-repo list).
  • agent-created — skills created by the curator / agent itself.
  • community — default fallback (URL installs, user copies).

The scanner has ~75 threat patterns covering env-var exfil (curl / wget / httpx / os.environ / process.env / ENV[]), prompt-injection ("ignore previous instructions", DAN mode, hidden HTML, role hijack), destructive ops (rm -rf, mkfs, dd, chmod 777, > /etc/), persistence (crontab, authorized_keys, systemd, .bashrc, AGENTS.md/CLAUDE.md/.cursorrules, SOUL.md), network (nc -l, ngrok, cloudflared, webhook-test services, pastebin), obfuscation (base64 -d | sh, eval/exec, atob/btoa, chr-building, unicode-escape chains), execution (subprocess.run, os.system, child_process.exec, Runtime.exec), path traversal (../../, /etc/passwd, /proc/self), crypto mining (xmrig, monero, stratum+tcp), supply-chain (curl-piped-to-sh, PEP 723, unpinned pip/npm, git clone, docker pull), privilege escalation (sudo, setuid, allowed-tools frontmatter), credential exposure (api_key=, ghp_, sk-, sk-ant-, AKIA, BEGIN PRIVATE KEY). 5.3 Dispatch flow (agent.py:288-397)

The single funnel is Agent._execute_tool_call_async. Four paths:

  1. set_mode → _handle_set_mode validates the target mode, requests user approval if the new mode has write-capable toolsets, sets self.mode on approve. 120s approval timeout via SSE.
  2. delegate_task → _handle_delegate_task emits a progress event, calls _dispatch_delegate_task which curates the subagent context, injects session context (sandbox, session_id, parent progress, delegate_depth+1), runs the 1700-line delegate_task.py subagent machinery, then pipes output through validate_subagent_output. This is the only path that recurses into a new agent loop.
  3. tool_search → _handle_tool_search runs the search, appends every returned spec to self._discovered_tool_schemas so the freshly-discovered tool is callable in the very next turn.
  4. Regular tool → 4-step gate-and-execute: (1) permissions.resolve_level, (2) deny short-circuit, (3) ask → 120s approval via SSE, (4) registry.execute with session_id / tool_call_id / sandbox_manager injected; output truncated via _apply_output_limit.

7.3 Search algorithm

Pure substring containment, no embeddings, no fuzzy matching, no tokenization, no stemming. Tier scoring (first match wins per node; not additive):

  • q in name → 3
  • q in summary / description → 2
  • q in tags → 1

Sort desc, take top N. Empty query → first 10 nodes verbatim. Returns {query, count, results: [nodes]}. For a 500-file KG with common substrings ("test", "src", "lib") the signal-to-noise is poor.

7.4 Generator (no AST, no symbol extraction)

build_graph(files, repo_url):

  1. For each path: extract extension, look up language label, append a node with summary = "File: <rel>".
  2. Aggregate lang_counts; primary = max(lang_counts) (or "" if all unknown).
  3. Emit {metadata, nodes, edges: []}.

Files without an extension or with unknown extensions get language="unknown" and empty tags. The UI/dashboard side of the contract accepts richer node shapes (function, class, interface with real summaries + multi-tag arrays) — the current generator does not produce those.

7.5 Dependency graph BFS

Regex-only. _IMPORT_RE captures three alternates: from X import …, import X, require('X') / require("X"). Walked extensions: *.{py, ts, tsx, js, jsx, rs, go} (7; despite kg_generator supporting 20). No multi-line / string-aware parsing. No module resolution. outgoing = BFS over import_map with visited set; incoming = direct reverse_map + transitive at depth > 1 (hard-capped 15 lines).
