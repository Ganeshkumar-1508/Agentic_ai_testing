## What it does

A coordinator-callable tool that re-indexes the knowledge graph so subsequent `codegraph_*` queries see the latest state. Replaces the simpler `KGRefreshTool` previously in `knowledge_graph_tool.py` with:

| Before | After (C04) |
|--------|-------------|
| Inline `codegraph sync` only | Calls `KnowledgeGraphSyncer.sync` (which runs sync + copies DB + writes provenance) |
| Returns new symbol/file count | Returns `{nodeCount, edgeCount, delta: {added, removed, modified}, duration_ms, last_refresh_at}` |
| No debounce | Hard debounce 60s, `force=true` bypass |
| No event emission | Emits `kg.refreshed` on EventBus / `stream_events` |
| No failure categorization | Categorized: `sync_failed` / `timeout` / `copy_failed` / `sandbox_unreachable`, with `recoverable` flag |
| No baseline | Orchestrator seeds baseline after the post-coordinator sync so the first in-run call has a real delta |

---

## The 12 locked decisions (from the decision tree)

1. **Trigger**: Manual via `kg_refresh()` tool call.
2. **Surface**: No args. Returns status + delta dict.
3. **Behavior**: Always incremental (codegraph auto-detects via mtime).
4. **Access**: Coordinator-only. The leaf-worker `allowed_leaf` set excludes `kg_refresh`.
5. **Return**: Status dict + delta vs last refresh.
6. **Debounce**: 60s hard limit, `force=true` escape. Configurable via `KG_REFRESH_DEBOUNCE_SECONDS`.
7. **Observability**: `stream_events` `kg.refreshed` event.
8. **End-of-run sync**: Keep (defense in depth).
9. **UI surface**: Header pill + dedicated panel (deferred to a future frontend change).
10. **Panel content**: Hero + aggregate + sparkline + filters.
11. **Failure model**: Categorized error re-raise.
12. **Implementation file**: `backend/harness/tools/kg_refresh_tool.py`.

## Implementation details

### `set_current_kg_context` / `get_current_kg_context`

Mirrors the existing `set_current_tracker` / `get_current_tracker` pattern. The orchestrator's `run_single` calls `set_current_kg_context(kg_ctx)` after building the context, and the tool reads it via `get_current_kg_context()`. Async-safe (ContextVar).

### `KgRefreshTool._seed_baseline(status)`

Called by the orchestrator's post-coordinator sync block. Seeds the tool's `_last_status` and `_last_refresh_at` so the first in-run `kg_refresh()` call computes a real delta against the just-finished state — not against the (potentially stale) initial index.

### `KgRefreshTool._compute_delta(fresh, prev)`

Count-based delta. `nodes_added = max(0, new - old)`, `nodes_removed = max(0, old - new)`, same for edges. `modified = 0` (limitation: codegraph -j only emits totals; per-file diff would need an extra `codegraph affected` call).

### `KgRefreshTool.run(**kwargs)`

The actual tool:
1. Debounce gate: if `last_refresh_at > 0` AND elapsed < debounce AND not `force=true`, return `{skipped: true, last_refresh_age_seconds: N, force_available: true}`.
2. Resolve sandbox + kg_ctx. If either is None, return `sandbox_unreachable` (recoverable: false).
3. Run `KnowledgeGraphSyncer.sync`. If it raises `asyncio.TimeoutError` → `timeout` (recoverable: true). If it raises anything else → `sandbox_unreachable` (recoverable: false). If it returns `{}` → `sync_failed` (recoverable: true).
4. Compute delta vs `_last_status`.
5. Update `_last_status`, `_last_refresh_at`.
6. Emit `kg.refreshed` event. If emit fails, log and continue (observability is non-fatal).
7. Return success result.

### Failure categories

| Category | When | Recoverable |
|----------|------|-------------|
| `sync_failed` | `KnowledgeGraphSyncer.sync` returned `{}` | true (mtime glitch — retry likely succeeds) |
| `timeout` | `asyncio.TimeoutError` from sync | true (sandbox may be slow — retry later) |
| `copy_failed` | (Future) DB copy to host fails | true (host disk issue) |
| `sandbox_unreachable` | No sandbox, no kg_ctx, sync raised unexpected | false (escalate to user) |

### Registration

Single registration in `coordinator` toolset. The `intelligence` toolset metadata is informational; the actual access control is at the role boundary in `delegate_task.py:312-317` (the `allowed_leaf` set excludes `kg_refresh`).

## Open questions (deferred)

1. Should `kg_refresh` also emit an L1 promotion event for the `L1Indexer`? Currently the L1Indexer extracts from tool calls, not from KG events.
2. Multi-repo behavior: each repo has its own KG. The tool uses the orchestrator's `kg_ctx`, which is per-run. The tool works for the primary repo only; context repos are out of scope.
3. Should the dashboard show KG staleness as a "warning" state when no refresh has happened in >5 minutes, even if the user hasn't requested one?
4. Interaction with the L2 memory pipeline (`l2_reflection.py`): does a `kg_refresh` count as an "agent work" event for the L2 curator?
5. Interaction with C03 (push-based completion): when the orchestrator receives a `subagent.completed` event for a leaf worker that wrote files, should it auto-trigger a `kg_refresh`? My current answer is no — the LLM decides.

## Usage example

The coordinator subagent calls:

```python
# First call: orchestrator's baseline is the seed. Delta may be 0.
result = kg_refresh()
# {
#   "nodeCount": 1234, "edgeCount": 5678,
#   "delta": {"nodes_added": 0, "nodes_removed": 0, "edges_added": 0, "edges_removed": 0, "modified": 0},
#   "duration_ms": 1240, "last_refresh_at": 1718975025.123,
#   "baseline_source": "kg_refresh"
# }

# After editing file X, the coordinator wants to verify the new symbols are indexed:
result = kg_refresh()
# {
#   "delta": {"nodes_added": 3, "nodes_removed": 0, "edges_added": 5, "edges_removed": 0, "modified": 0},
#   ...
# }

# Second call within 60s — debounce skips it:
result = kg_refresh()
# {
#   "skipped": true,
#   "last_refresh_age_seconds": 23,
#   "force_available": true
# }

# To verify a specific edit took effect:
result = kg_refresh(force=true)
# ...

# On sandbox failure:
result = kg_refresh()
# success=False, error="sandbox_unreachable", data={category: "sandbox_unreachable", recoverable: false, ...}
```
